\begin{frame}
\frametitle{Overview}

\begin{itemize}
    \item Transformer networks
    \begin{itemize}
        \item Type of neural network architecture that is trained to continue a sequence of integers
        \item These can represent characters (ASCII), words, tokens (parts of words)
        \item Introduced in \textit{Attention Is All You Need [Vaswani et al., 2017]}
        \item Initially introduced for natural language translation tasks
        \item Later employed successfully for all sorts of tasks, including as a general-purpose AI agent
        \item Transformer networks power state of the art AI systems (e.g., ChatGPT, various coding agents)
    \end{itemize}
    \item Integer arithmetic
    \begin{itemize}
        \item Addition \texttt{123+456=579.}
        \item Multiplication \texttt{456*789=359784.}
        \item Integer square root \texttt{123:11.}
    \end{itemize}
    \item Goal is to have the model complete the answer (arithmetic result)
    based on the operands: Model gets \texttt{123+456=} and should continue with the result
    \texttt{579}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Why even do this?}
\begin{itemize}
    \item No need to use transformers or even any machine learning model for simple aritmetics
    \item Nonetheless, it is interesting to see capabilities of models to learn based on examples alone
    \item Only one correct result, no ambiguity, easy verification
    \item Multiplication in particular is hard for even (in 2023) state-of-the-art general-purpose transformer models (GPT-3.5, GPT-4)
    \item To be fair, current models can often solve such tasks by calling an external tool or step-by-step reasoning
    \item Differing theoretical complexities per operation: $\operatorname{O}(n)$ for addition, $\operatorname{O}(n^2)$ for naive multiplication
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Existing work}
\begin{itemize}
    \item \textit{Teaching Arithmetic to Small Transformers [Lee et al., 2024]}
    \begin{itemize}
        \item train small transformer networks on 3-digit up to 10-digit addition
        \item plotted final accuracy in response to training dataset size
        \item tested the influence of various strategies like result reversal and step-by-step training data
        \item also tried multiplication, but only 2-digit operands
    \end{itemize}
    \item \textit{Goat: Fine-tuned LLaMA Outperforms GPT-4 [Liu and Low, 2024]}
    \begin{itemize}
        \item larger transformer network (7 billion parameters)
        \item not training from scratch, use existing open-weight language model base
        \item up to 16 digit operands, addition and multiplication, step-by-step training data
        \item model learned addition perfectly, but failed at multiplication
    \end{itemize}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Initial experiments}
\begin{itemize}
    \item Three integer arithmetic operations:
    \begin{itemize}
        \item addition, multiplication, integer square root
    \end{itemize}
    \item Three operand sizes:
    \begin{itemize}
        \item 3, 5, 10 digits for addition and multiplication
        \item 6, 10, 20 digits for integer square root
    \end{itemize}
    \item Three dataset sizes:
    \begin{itemize}
        \item 1k, 10k, 100k elements
    \end{itemize}
    \item 27 experiments in total
    \item Small transformer network:
    \begin{itemize}
        \item OpenAI GPT2 variant (released in 2019)
        \item trained from scratch on a single task only
        \item about 10M parameters
        \item ASCII tokenization (one token per character)
    \end{itemize}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Further experiments}
\begin{itemize}
    \item Changing model size
    \item Reversing the result
    \begin{itemize}
        \item \texttt{123+456=579} becomes \texttt{123+456=975}
        \item Order of digits then matches order of computation
    \end{itemize}
    \item Changing sample distribution
    \begin{itemize}
        \item Uniform digits, uniform bits...
        \item Shifts balance between "easy" and "hard" samples
    \end{itemize}
    \item Step-by-step computation examples ("scratchpads")
    \begin{itemize}
        \item Encode intermediate steps into training samples, not just the result
        \item For example, for addition include steps for each digit, carry propagation...
    \end{itemize}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Conclusion}

\begin{itemize}
\item Going from 99\% to 100\% accuracy can still be an issue (seen when learning addition without scratchpad)
\item Transformer networks can learn addition (directly) and also multiplication (with help from scratchpad)
\item Small transformers can learn the structure of long scratchpads
\item But the scratchpad needs to be in small steps (speculated cause for issues seen with integer square root)
\item How to compute the result must be known to generate a scratchpad
\item Scratchpads have downsides of longer training sequences (slower training, higher memory demand)
\end{itemize}
\end{frame}
